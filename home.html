Simple Eagerly Singleton

The fact that the field is "static" and initialized as a "new object" means that he will be loaded in the very first steps of the program loading - what makes him eagerly.
Constructor must be private.
Getter must be public because that's the only way to get the instance outside of that particular class.

Lazily Singleton But Not a Thread Safe

Compared to the first example, instead of generating a "new object" in the field, we will initialize the field value to null.
This way, "new object" will be created only by demand.
The instance will be created within the "getter" method only once, at the first time an external entity outside of this specific class requires it.
